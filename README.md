# NanoScreen-AI Tumor Delivery Prioritization

## Project Title

Artificial intelligence-assisted prioritization of nanoparticle formulations for efficient tumor delivery

## Overview

This repository is prepared as the paper code and supplementary-file package for a screening-oriented machine-learning framework that prioritizes nanoparticle formulations with high predicted tumor delivery efficiency. The workflow includes dataset curation, high-delivery label construction, model optimization, out-of-fold and independent test-set evaluation, SHAP-based interpretation, virtual formulation screening, cancer-type-specific candidate screening, and local working-range summarization.

Model-prioritized candidates are computational outputs intended to guide future experimental validation. They are not experimentally validated optimal formulations.

## Public Scripts

The `scripts/` folder contains cleaned public code records curated from the project notebooks and supplementary-file generation scripts.

- `main_analysis_final.py` is the manually organized final public analysis code exported from the finalized Jupyter workflow. It preserves the main analysis workflow, including preprocessing, group-aware splitting, label construction, model training and evaluation, OOF-based model selection, SHAP analysis, virtual candidate screening, cancer-type-specific screening, and local working-range estimation.
- `figure_generation_cleaned.py` is a cleaned public code record exported from the figure-generation notebook. It currently preserves code for selected data-driven performance-panel generation, especially Figure 4-related performance panels.

SHAP plotting and SHAP export logic are preserved mainly in `main_analysis_final.py` rather than `figure_generation_cleaned.py`.

Figure 1 was prepared separately as a schematic workflow figure and is not generated by the Python scripts. `figure_generation_cleaned.py` should not be interpreted as reproducing all final manuscript figures.

Running the cleaned scripts may train models, recompute outputs, or regenerate figures. They were not executed during repository preparation. Manual review, dependency checks, and path/input adaptation may be required before full re-execution in a different environment.

## Supplementary Files

The `supplementary/` folder contains the final supplementary materials prepared for the manuscript:

- `Supplementary_Material.docx`: supplementary tables and supplementary data/code availability statement.
- `Supplementary_Excel_File_A_dataset_and_labels.xlsx`: cleaned dataset, split labels, endpoint definitions, threshold sensitivity analysis, variable dictionary, variable ranges, and categorical levels.
- `Supplementary_Excel_File_B_model_evaluation_outputs.xlsx`: model-evaluation outputs, cross-validation results, out-of-fold and independent test-set predictions, model reports, optimized hyperparameters, and ranking-oriented metrics.
- `Supplementary_Excel_File_C_virtual_screening_outputs.xlsx`: virtual formulation screening outputs, scored candidates, top-ranked candidates, cancer-type-specific candidates, and local working-range outputs.

The supplementary Excel and Word generation scripts in `scripts/` are retained for supplementary-file organization:

- `build_supplementary_excel_A.py`
- `build_supplementary_excel_B.py`
- `build_supplementary_excel_C.py`
- `build_supplementary_material_docx.py`

## Data Source

The analytical dataset and data dictionary used for the paper are provided in Supplementary Excel File A. The file contains the cleaned analytical dataset, training and test split, high-delivery labels, endpoint definition, threshold sensitivity analysis, variable dictionary, variable ranges, and categorical levels.
